# @evano/website

The official Evano Studio website for **evanostudio.com** — built with **Next.js (App Router) + React + TypeScript**.

> 🚧 Evano Studio is in **alpha**. This site is intentionally honest about that: there is no public download yet, and links to GitHub/downloads are placeholders until real releases exist.

## Tech

- **Next.js** (App Router) + **React** + **TypeScript**
- **No CSS framework** — a small, dependency-free design system in `src/app/globals.css`
- **No tracking / analytics** by default (privacy-first, like the product)
- SEO-friendly metadata, `robots.txt`, and `sitemap.xml` generated by Next.js

## Requirements

- Node.js ≥ 20
- pnpm ≥ 9 (this app is part of the repo's pnpm workspace)

## Run it locally

From the **repository root**:

```bash
pnpm install            # installs workspace dependencies
pnpm dev:website        # starts the website dev server
```

Or from **this folder** (`apps/website`):

```bash
pnpm dev                # http://localhost:3000
```

## Other commands

```bash
pnpm build              # production build
pnpm start              # serve the production build (after build)
pnpm typecheck          # TypeScript type checking (tsc --noEmit)
```

## Editing content

Most marketing copy lives in **`src/content/site.ts`** (site name, tagline, features, FAQ, tech stack, placeholder links). The documentation guides live in **`src/content/docs.ts`** as structured blocks (no MDX), rendered by `components/docs/DocBody.tsx` and routed via `app/docs/[slug]`. Edit there to add or update guides. Longer page-specific prose lives in each page under `src/app/*`.

## Deployment (evanostudio.com)

The site is a standard Next.js app and can be deployed to any Next-compatible host (e.g. Vercel) by pointing it at `apps/website`. Before public launch, replace the placeholder links in `src/content/site.ts` (`links.github`, `links.download`) with real URLs and review all copy (see `docs/WEBSITE.md` for the honest-marketing rules).
